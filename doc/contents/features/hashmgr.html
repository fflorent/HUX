Hash Manager

The Hash Manager is a plugin for HUX that updates the Hash of your URL at the same time that you load contents.

This plugin is recommended when you need to store in your URL the contents that are injected via HUX in your document for most of the browsers.

NOTE : This plugin can only replace the content of a target element (no append nor prepend of content).

How To Use it

You can simply use it by creating links of this syntax :

<a href="#!TARGET_ID=URL_OF_YOUR_CONTENT">TEXT</a>

Where :

  • TARGET_ID is the ID of your target element (see target to understand what it is);
  • URL_OF_YOUR_CONTENT is the URL of the content you want to load;

WARNING : HUX does not support cross-domain HTTP Requests. This means that you can only load content of your own site.

We name a pair of the Hash any pair containing an target ID and a URL. The key is the ID preceded by an exclamation mark ("!") and the value is the URL.

There are some rules around the management of the pairs :

  1. any key must begin with an exclamation mark ("!"), and what follows is the target id;
  2. the pairs are comma-seperated, for example : #!target1=url1,!target2=url2,... ;
  3. if there is two (or more) pairs having the same key (i.e. target id), the newest one replaces the old one(s)
  4. if a target id in a key references a non-existant element, the pair is removed from the hash
  5. clicking on a link having href="#!TARGET_ID=URL" makes append the value of this href in the hash
  6. the original content of a target is stored as the default content before its replacement
  7. if you remove any pair in the hash, the default content of its target is loaded
  8. if you set the value of a pair to "__default", that removes the pair from the Hash, and the default content is loaded
  9. if a pair is present twice or more in the hash, what follows the first redundant pair is removed :
    so, for example, if you click on this link ("#!container=features"), you will go to the home page of features

Look at the examples to assimilate each of these rules

Examples

Example 1 : the target

Code :

<a href="#!ex_hm_target1=resources/hello_world.html">click to run Example 1</a>
<div id="ex_hm_target1">I will disapear...</div>

Rendering :

click to run Example 1
I will disapear...

Example 2 : multiple pairs

Code :

<a href="#!ex_hm_target2.1=resources/hello_world.html,!ex_hm_target2.2=resources/how_are_you.html">click to run Example 2</a>
<div id="ex_hm_target2.1">I will disapear...</div>
<div id="ex_hm_target2.2">I will disapear too...</div>

Rendering :

click to run Example 1
I will disapear...
I will disapear too...
