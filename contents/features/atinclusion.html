At Inclusion

Warning : At Inclusion requires Firefox 4+, Chrome 5+, Opera 11.50+, and Safari5+
To support other browsers/versions, you are recommended to use At Inclusion Fallback

This module is still experimental. See known bugs

At Inclusion is a module for HUX that manages history and updates URL as the user loads contents.

By updating URL, At Inclusion ensures that the loaded contents are retrieved when the page is refreshed, or if the URL is shared with someone else.

By managing history, At Inclusion lets the user click on "Back"/"Next" button of their browser to navigate through the history.

Differences with Hash Bang

At Inclusion uses the last history API (pushstate, replacestate) available for Chrome 5+, Firefox 4+, Internet Explorer 10+, Opera 11.5+ and Safari 5.0+

For the older browsers, you are proposed to use Hash Bang coupled with AtInclusions fallback

This table compares the advantages of At Inclusion and Hash Bang :

Advantage Hash Bang At Inclusion
URL Update yes yes
History management (go backward/forward) yes yes
Old browsers compatibility (including IE7) yes yes 1
Indexing (Google, Bing, Yahoo...) no yes with atinclusions Apache module 2

1 : needs At Inclusion Fallback and Hash Bang
2 : still experimental


If you need to Ajaxise your website, but you also want to index it, you are highly recommended to use At Inclusion with At Inclusion Fallback for older browsers.

If you use your website for a specific browser, for example in an intranet, you can consider using either At Inclusion or Hash Bang, depending on the compatibilities.

Known Bugs

We can consider the fact that letting the user load contents in any target as a security issue, even though that contents are provided by HTML pages hosted in the same server.

Until a solution is provided, you are recommended not to use Hash Bang nor At Inclusion except for test purposes and with pages hosted by a test server.
